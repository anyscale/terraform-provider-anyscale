release-note:breaking-change
resource/anyscale_cloud: `file_storage.mount_targets` changes from a block to a list-of-objects attribute; rewrite `mount_targets { address = "..." zone = "..." }` as `mount_targets = [{ address = "...", zone = "..." }]`. The value is now Computed and self-heals the same way as the memorydb/memorystore fix for clouds imported between v0.15.2 and v0.16.0 inclusive; clouds imported with v0.16.1 or later keep their null value until re-imported. Existing state auto-migrates to the new schema on your next plan alongside the kubernetes_config change below; no manual state surgery is needed.

release-note:breaking-change
resource/anyscale_cloud: `kubernetes_config.namespace`, `ingress_host`, `cluster_name`, `context`, and `kubeconfig_path` are removed; they were deprecated since v0.1.2 and already had no effect (never sent to the Anyscale API). To migrate, delete these five attributes from your configuration — `terraform plan`/`validate` errors with "Unsupported argument" if any remain. Existing state auto-migrates to the new schema on your next plan; no manual state surgery is needed, including for clouds that never set any of the five fields.
